# 听力提高练习助手 — 产品功能规格 v1.0

## 一、产品定位

桌面端英语听力辅助练习生成器。针对 KET / PET / 中考 / 高考 / 雅思 五个考试级别，覆盖四种薄弱类型，利用大模型 + TTS 生成针对性听力训练材料，本地化管理学生与练习记录。

---

## 二、核心业务流程

```
学生姓名 + 问题句(按薄弱类型) + 考试类型 + 薄弱类型(可多选)
 ↓
 DeepSeek 分析诊断
 ↓
 ┌───────────┼───────────┐
 ↓ ↓ ↓
 1. 诊断分析 2. 生成练习句 3. 生成音频
 (为什么没听懂) (3/5/10/自定义) (TTS朗读)
 ↓
 保存到本地数据库
 (学生+练习+音频)
```

## 三、功能模块

### 模块 1：练习生成（核心）

#### 1.1 输入区

| 字段 | 组件类型 | 说明 |
|:----|:--------|:-----|
| 学生姓名 | 下拉输入框 | 可输入新姓名，也可从历史记录下拉选择 |
| 考试类型 | 下拉框 | KET / PET / 中考 / 高考 / 雅思 |
| 薄弱类型 | 多选卡片/按钮 | 单词发音 / 固定搭配 / 句式结构 / 言外之意，支持多选 |
| 问题句输入 | 动态表单 | 按薄弱类型切换不同输入字段（见 1.2） |
| 练习句数量 | 选择器 | 3 / 5 / 10 / 自定义输入 |
| 生成按钮 | Button | 生成练习 |

#### 1.2 动态输入表单（按薄弱类型切换）

##### 单词发音听不懂
| 字段 | 组件 | 提示语 |
|:----|:----|:-------|
| 原句 | textarea | 请输入完整的原文句子 |
| 没听清的单词 | input | 哪个词/哪部分发音没抓住？(可选填) |

##### 固定搭配听不出来
| 字段 | 组件 | 提示语 |
|:----|:----|:-------|
| 原句 | textarea | 请输入完整的原文句子 |
| 感觉有搭配的位置 | textarea | 大概哪里觉得有固定搭配但没听出来？(可选填) |

##### 句式结构没听懂
| 字段 | 组件 | 提示语 |
|:----|:----|:-------|
| 原句 | textarea | 请输入完整的原文句子 |
| 困惑的部分 | textarea | 哪个结构没理解？（倒装/从句/省略/虚拟语气…）(可选填) |

##### 言外之意未获取到
| 字段 | 组件 | 提示语 |
|:----|:----|:-------|
| 上下文/对话场景 | textarea | 谁在说话？什么情境？(可选填) |
| 原句 | textarea | 具体的对话内容 |

> **注意：** 多选薄弱类型时，输入框取并集——即显示所有选中类型对应的字段，但"原句"字段共用。

#### 1.3 输出区

##### 诊断分析
- AI 对问题句的详细解析
- 标注难点（发音/搭配/结构/语用）
- 关联数据库中对应级别的词汇/搭配

##### 针对性练习列表
- 每个练习句包含：
 - ▶️ 播放按钮
 - 句子原文
 - 参考答案（可折叠展开）
 - 解析说明

##### 音频控制
- 每个句子独立播放
- 语速调节：正常 / 慢速（0.75x）
- 单句循环 / 全部连续播放
- 音频自动保存到本地

### 模块 3：错误处理

#### 3.1 DeepSeek API 错误

| 错误场景 | 用户看到的提示 | 引导建议 |
|:--------|:-------------|:---------|
| API Key 未配置 | ❌ API密钥未设置 | 请前往 设置 → API配置 输入您的 DeepSeek API Key |
| API Key 无效 | ❌ API密钥无效 | 请检查您的 DeepSeek API Key 是否正确，可前往 设置 重新输入 |
| 网络连接失败 | ❌ 网络连接失败 | 请检查您的网络连接，或确认 DeepSeek API 地址是否可以访问 |
| API 请求超时 | ❌ 请求超时 | 服务器响应较慢，请点击"重试"重新生成 |
| API 返回格式异常 | ❌ 返回数据异常 | 生成的内容格式有误，请点击"重试"重新生成 |
| API 余额不足/频率限制 | ❌ API调用受限 | DeepSeek 账户可能余额不足或触发频率限制，请检查账户状态 |
| 服务器内部错误(5xx) | ❌ 服务器暂时不可用 | DeepSeek 服务器暂时异常，请稍后重试 |

#### 3.2 TTS 语音合成错误

| 错误场景 | 用户看到的提示 | 引导建议 |
|:--------|:-------------|:---------|
| Edge TTS 网络失败 | ⚠️ 语音生成失败 | 网络连接异常导致无法合成语音，请检查网络后重试 |
| 文本过长无法合成 | ⚠️ 文本超出长度限制 | 该句子过长无法生成语音，可尝试缩短后重试 |

#### 3.3 输入验证

| 场景 | 处理方式 |
|:----|:--------|
| 学生姓名为空 | 生成按钮禁用，hover提示"请输入学生姓名" |
| 原句为空 | 生成按钮禁用，hover提示"请输入原句" |
| 考试类型未选 | 生成按钮禁用，hover提示"请选择考试类型" |
| 薄弱类型未选 | 生成按钮禁用，hover提示"请至少选择一个薄弱类型" |
| 练习句数量自定义输入非数字 | 输入框边框变红，提示"请输入有效数字" |
| 练习句数量超出范围（<1或>50） | 输入框边框变红，提示"数量范围为1-50" |

```
┌──────────────────────────────────────────────┐
│ 选择保存位置 │
│ │
│ 此电脑 > 本地磁盘(D:) > 听力资料 │
│ ├── PET │
│ │ ├── 张三 │
│ │ ├── 李四 │

│ ├── KET │

│ ├── 高考 │
│ └── 雅思 │
│ │

│ │
│[新建文件夹][确定][取消] │
└──────────────────────────────────────────────┘
```

- 首次使用时默认定位到 `{用户文档目录}/ListeningTrainer/{等级}/{学生}/`
- 之后自动记住用户上次选择的路径
- 用户可直接点"确定"接受默认位置
- 也可在树形目录中自行浏览选择或新建文件夹

**第2步**：确认后，程序自动在选择的目录下创建日期子目录：

```
自动在 {用户选择的目录}/ 下创建:

```

最终完整路径为：
```
D:\听力资料\PET\张三\2026-06-22_发音+搭配\
```

> 用户只需要关心前三级目录（根目录/等级/学生），第四级（日期_类型）由程序自动创建。

#### 5.3 每次生成保存的文件

每次生成在叶子目录下保存以下文件：

| 文件 | 格式 | 说明 |
|:----|:----|:------|
| `01_{英文关键词}.mp3` | MP3 | 第1句音频 |
| `01_句子解析.txt` | UTF-8 TXT | 第1句的文本+翻译+解析 |
| `02_{英文关键词}.mp3` | MP3 | 第2句音频 |
| `02_句子解析.txt` | UTF-8 TXT | 第2句的文本+翻译+解析 |
| ... | ... | 以此类推 |
| `诊断分析.txt` | UTF-8 TXT | AI 对原句的诊断分析（纯文本，方便打开阅读） |
| `session.json` | JSON | 本次生成的完整结构化数据（供软件内部读取恢复） |

**解析文件（.txt）内容格式示例：**
```
句子: Could you possibly pass me the salt?
翻译: 你能把盐递给我吗？
类型: 单词发音

难点词: possibly /ˈpɒsəbli/
发音现象: 弱读 — possibly 中的 'o' 弱读为 /ə/
解析: 在快速语流中，possibly 的第二个音节被大幅度弱化...

同类练习:
 原句: Could you possibly help me with this?
 练习: Could you possibly pass me the salt?
```

> 用 `.txt` 而非 `.docx` 或 `.pdf`，保证任何设备都能直接打开查看，无需额外软件。

## 四、UI 视觉设计规范

### 4.1 设计哲学

**「纸墨意境 · Scholarly Warmth」**

设计灵感来源于"纸墨"——纸张的温润质感与墨色的沉稳深邃。整个界面像一本精心编排的练习册，让学习者在数字环境中感受到纸质学习材料的亲近感。

| 关键词 | 诠释 |
|:------|:-----|
| 温润 | 暖色基底、柔和光影，不冰冷 |
| 沉稳 | 墨色为主、色彩克制，不浮躁 |
| 有序 | 清晰的信息层级、一致的节奏 |
| 专注 | 去除装饰干扰，内容即界面 |

### 4.2 桌面应用设计原则

本产品为 Windows 桌面应用（PySide6），所有交互遵循桌面平台规范，而非 Web 或移动端模式。

| 原则 | 说明 |
|:----|:-----|
| **标准窗口** | 使用系统原生标题栏（标题/最小化/最大化/关闭），不自绘标题栏 |
| **面板滚动** | 内容在面板内滚动，非整页滚动。左侧学生列表、右侧内容区各自独立滚动 |
| **选中即焦点** | 点击选中项以高亮条标记（非 Web 的 outline 边框） |
| **右键菜单** | 列表项支持右键上下文菜单（删除存档、打开所在文件夹等） |
| **键盘操作** | Tab 键在控件间切换焦点，Enter 确认，Esc 取消/关闭弹窗 |
| **快捷键** | Ctrl+Enter 快速生成，F5 刷新，Delete 删除选中项 |
| **原生对话框** | 文件选择使用 `QFileDialog`（系统原生），不自制文件选择器 |
| **拖拽操作** | 列表行可拖拽排序，窗口可拖拽调整大小 |
| **信息密度** | 桌面应用比 Web 有更高的信息密度，合理利用空间 |
| **持久状态** | 记住窗口大小位置、上次选择的路径、展开/折叠状态 |

### 4.5 间距与圆角

```
间距体系 (4px 基准):
 2 → 8px 组件内间距（小）
 3 → 12px 组件内间距（中） 
 4 → 16px 卡片内边距
 5 → 20px 组件间距
 6 → 24px 区块间距
 8 → 32px 大区块间距
 10 → 40px 页面边距

圆角体系:
 sm 4px 标签、chip、小元素
 md 8px 按钮、输入框
 lg 12px 卡片、面板、弹窗
 xl 16px 大卡片、模态框
 full 999px 头像圈、开关
```

### 4.6 层级（Z-order）

桌面应用通过**边框 + 背景色差**区分层级，不依赖大面积阴影。

```
层级0  底层面板    背景 #F5F0E8, 无边框
层级1  表面面板    背景 #FFFFFF, 1px 边框 #D8D2C8
层级2  弹窗/覆盖层  背景 #FFFFFF, 1px 边框 #C4BEB4, shadow 0 2px 8px rgba(44,40,32,0.08)
层级3  全屏遮罩     半透明黑 overlay, 顶层内容白底 + 边框
```

> 桌面应用不使用 web 式的大面积卡片阴影（Material Design 风格）。现代 Windows 设计规范（WinUI / Fluent Design）使用**边框区分**而非阴影。

## 五、数据库设计（供 LLM 检索使用）

### 5.1 考试等级说明表（`level_info`）

为 LLM 提供各等级的语境说明，便于生成适配难度的内容。

| 等级 | 全称 | 词汇量 | 对应CEFR | 说明 |
|:----|:-----|:------|:---------|:-----|
| KET | Key English Test | ~700 | A2 | 基础生存英语，日常话题（家庭、学校、食物、交通） |
| PET | Preliminary English Test | ~2,600 | B1 | 中级日常英语，能应对旅游、办公、社交场景 |
| 中考 | 中考英语 | ~1,600 | A2-B1 | 国内初中课标，话题偏校园、生活、传统文化 |
| 高考 | 高考英语 | ~4,800 | B2 | 国内高中课标，涵盖科技、环保、社会议题等较抽象话题 |
| 雅思 | IELTS | ~1,700(听力场景) | B2-C1 | 学术/留学场景，偏校园生活、学术讨论、公共服务 |

### 5.2 数据表结构与说明

#### `words` — 核心词汇表（11,320 条）

| 字段 | 类型 | 用途（对 LLM） |
|:----|:----|:--------------|
| id | int | 主键 |
| word | text | **单词本身**，用于匹配用户输入句中的词汇 |
| phonetic | text | 音标，用于发音指导 |
| pos | text | 词性（n./v./adj./adv.等） |
| meaning | text | **中文释义** |
| level | text | **所属等级**（KET/PET/中考/高考/雅思），用于限制生成范围 |
| source | text | 来源（如"PET单词表汇总"） |
| source_detail | text | 详细来源信息（如"译林版B1-U3"、"高考required/elective"） |
| category | text | 分类（如雅思场景分类"求职/课程入会"） |

> **LLM 使用方式**：用户输入句子后，提取句中单词查此表，确定哪些词在当前等级范围内、哪些超纲，据此调整练习句的用词难度。

#### `collocations` — 固定搭配表（794 条）

| 字段 | 类型 | 用途 |
|:----|:----|:-----|
| id | int | 主键 |
| collocation | text | **搭配本身**（如"turn off"、"contribute to"） |
| meaning | text | 中文释义 |
| phonetic | text | 音标（如果有） |
| level | text | 所属等级 |
| source | text | 来源 |
| source_detail | text | 详细来源 |
| category | text | 场景分类 |
| example | text | 例句（如果有） |

> **LLM 使用方式**：当用户在"固定搭配"薄弱类型下输入句子时，匹配此表找出句中涉及的搭配，生成同类搭配的替换练习。

#### `exercises` — 言外之意题库（258 道）

| 字段 | 类型 | 用途 |
|:----|:----|:-----|
| id | int | 主键 |
| level | text | 等级 |
| problem_type_id | int | 对应薄弱类型（=4 言外之意） |
| category | text | **语用分类**（如"间接请求"、"隐含否定"、"含蓄拒绝"） |
| difficulty | text | 难度（★☆☆/★★☆/★★★） |
| source | text | **真题来源**（如"CET IELTS 10, Test 3"） |
| dialogue | text | **对话原文** |
| literal_q | text | 字面理解问题 |
| literal_options | text | 字面题选项（A-D） |
| pragmatic_clues | text | **语用线索分析**（如何推断言外之意） |
| implied_q | text | 隐含意义问题 |
| implied_options | text | 隐含题选项（A-D） |

> **LLM 使用方式**：作为 few-shot 示例。当生成言外之意练习时，从此表按等级和分类检索同类题目作为参考模板。

#### `problem_types` — 薄弱类型表（4 种）

| id | name | label | description |
|:---|:-----|:------|:------------|
| 1 | pronunciation | 单词发音听不懂 | 无法听清单词的发音，尤其是连读、弱读、同化等语音现象 |
| 2 | collocation | 固定搭配听不出来 | 无法识别常见的固定搭配、短语动词、习惯表达 |
| 3 | structure | 句式结构没听懂 | 对复杂句式结构理解困难，如倒装、从句、省略等 |
| 4 | implicature | 言外之意未获取到 | 无法理解说话人的隐含意图、言外之意、语用含义 |

#### `students` — 学生表

| 字段 | 类型 | 用途 |
|:----|:----|:-----|
| id | int | 主键 |
| name | text | 学生姓名（唯一） |
| created_at | timestamp | 创建时间 |

#### `practice_records` — 练习记录表

| 字段 | 类型 | 用途 |
|:----|:----|:-----|
| id | int | 主键 |
| student_id | int | 关联学生 |
| exercise_id | int | 关联题库习题（为null表示 AI 生成） |
| level | text | 练习时的考试等级 |
| problem_type_id | int | 薄弱类型 |
| input_sentence | text | 用户输入的原句 |
| input_details | text | 用户输入的补充信息（JSON） |
| output_json | text | AI 生成的完整结果（JSON） |
| audio_dir | text | 音频文件目录路径 |
| score | int | 评分（预留） |
| created_at | timestamp | 练习时间 |

## 六、AI 提示词设计

### 6.1 总体策略

每个生成请求分为**三个阶段**，分别调用 LLM：

```
阶段1：诊断分析
 输入：用户句子 + 等级 + 薄弱类型 + 数据库词汇上下文
 输出：诊断JSON

阶段2：生成练习
 输入：诊断结果 + 等级词汇范围 + 同类习题示例(few-shot) + 练习数量
 输出：练习句列表JSON

阶段3：TTS 语音合成（不经过 LLM，直接调用 TTS 引擎）
```

### 6.2 数据库上下文注入

每次生成前，系统自动执行以下查询，作为 Prompt 的上下文：

```sql
-- 1. 获取该等级词汇范围（用于限制定级用词）
SELECT word, meaning FROM words WHERE level = '{等级}';

-- 2. 如果用户句子包含已知词汇，标记其难度
SELECT word, level FROM words 
WHERE word IN ('提取自用户句子的单词列表');

-- 3. 如果薄弱类型是"言外之意"，获取同类 few-shot 示例
SELECT category, dialogue, pragmatic_clues, implied_q, implied_options
FROM exercises 
WHERE level = '{等级}' AND problem_type_id = 4
ORDER BY RANDOM() LIMIT 3;

-- 4. 如果薄弱类型是"固定搭配"，获取相关搭配
SELECT collocation, meaning FROM collocations 
WHERE level = '{等级}' ORDER BY RANDOM() LIMIT 10;
```

### 6.3 各薄弱类型的 Prompt 模板

#### 单词发音听不懂

**System Prompt:**
```
你是一位英语听力教师，擅长标注和分析发音难点。
等级限制：{等级}（词汇范围限于该等级词表）
任务：分析用户提供的句子，找出可能导致听力困难的发音现象，生成同类发音练习。

发音难点分析维度：
1. 连读(liaison)：如 "an apple" → /əˈnæpl/
2. 弱读(weak form)：如 "I can do it" → /aɪ kən duː ɪt/
3. 同化(assimilation)：如 "don't you" → /ˈdəʊntʃuː/
4. 省音(elision)：如 "government" → /ˈɡʌvnmənt/
5. 易混音对：如 /θ/ vs /s/, /l/ vs /n/ 等
```

**输出 JSON 格式：**
```json
{
 "diagnosis": {
 "original_sentence": "用户输入的原句",
 "difficult_words":[
 {
 "word": "发音难点词",
 "phonetic": "音标",
 "phenomenon": "连读/弱读/同化/省音/易混",
 "explanation": "为什么这个词的发音容易听错"
 }
],
 "summary": "整体诊断总结"
 },
 "exercises":[
 {
 "sentence": "包含同类发音现象的练习句",
 "target_word": "练习目标词",
 "phonetic": "音标",
 "phenomenon": "发音现象说明",
 "translation": "中文翻译",
 "audio_text": "用于TTS朗读的文本(与sentence相同)"
 }
]
}
```

#### 固定搭配听不出来

**System Prompt:**
```
你是一位英语听力教师，擅长识别和讲解固定搭配。
等级限制：{等级}
任务：分析用户句子中涉及的固定搭配、短语动词、习惯表达，生成同类搭配练习。

搭配类型：
1. 动词短语(phrasal verbs)：如 "give up", "look after"
2. 介词搭配：如 "interested in", "depend on"
3. 习惯表达：如 "by the way", "as a matter of fact"
4. 固定句式：如 "would rather...than...", "not only...but also..."
```

**输出 JSON 格式：**
```json
{
 "diagnosis": {
 "original_sentence": "用户输入的原句",
 "key_collocations":[
 {
 "collocation": "搭配",
 "meaning": "释义",
 "type": "动词短语/介词搭配/习惯表达/固定句式"
 }
],
 "summary": "诊断总结"
 },
 "exercises":[
 {
 "sentence": "包含同一搭配的练习句",
 "target_collocation": "练习目标搭配",
 "meaning": "释义",
 "translation": "中文翻译",
 "audio_text": "TTS朗读文本"
 }
]
}
```

#### 句式结构没听懂

**System Prompt:**
```
你是一位英语语法教师，擅长拆解复杂句式结构。
等级限制：{等级}
任务：分析用户句子中包含的句式结构，通过句子成分拆解帮助学生理解，生成同类结构练习。

句式结构类型（按等级递进）：
- KET/PET：简单句、并列句、基础复合句（because/if/when）
- 中考：宾语从句、定语从句、状语从句、被动语态
- 高考：非谓语动词、虚拟语气、倒装、强调句、名词性从句
- 雅思：复杂嵌套从句、条件句混合时态、学术句式

输出应包括：句子成分标注（主/谓/宾/定/状/补）、结构图解、同类替换练习。
```

**输出 JSON 格式：**
```json
{
 "diagnosis": {
 "original_sentence": "用户输入的原句",
 "structure_breakdown": {
 "pattern": "句式结构名称",
 "explanation": "结构解析",
 "subject": "主语",
 "predicate": "谓语",
 "object": "宾语/表语",
 "modifier": "修饰成分"
 },
 "summary": "诊断总结"
 },
 "exercises":[
 {
 "sentence": "同结构的练习句",
 "pattern": "结构名称",
 "structure_breakdown": "简要结构拆解",
 "translation": "中文翻译",
 "audio_text": "TTS朗读文本"
 }
]
}
```

#### 言外之意未获取到

**System Prompt:**
```
你是一位英语语用学教师，擅长分析和讲解对话中的隐含意义。
等级限制：{等级}
参考题库：{从exercises表检索的同类few-shot示例}

任务：分析用户提供的对话场景和句子，解释其中隐含的言外之意，然后生成同类对话练习。

语用策略类型（参考数据库分类）：
- 间接请求：通过描述需求而非直接请求
- 隐含否定：先肯定再转折
- 含蓄拒绝：找借口而非直接拒绝
- 隐含态度：通过语气和选词透露态度
- 礼貌策略：通过 hedging 弱化直接性
- 弦外之音：情感潜台词
```

**输出 JSON 格式：**
```json
{
 "diagnosis": {

 "analysis": {
 "literal_meaning": "字面意思",
 "implied_meaning": "隐含意思",
 "pragmatic_clues": "语用线索分析",
 "strategy": "使用的语用策略"
 },
 "summary": "诊断总结"
 },
 "exercises":[
 {
 "context": "对话场景/上下文",
 "dialogue": "包含同类语用策略的对话",
 "question": "理解问题",
 "options":["A.选项1", "B.选项2", "C.选项3", "D.选项4"],
 "answer": "正确答案",
 "pragmatic_clues": "语用线索提示",
 "audio_text": "对话原文（用于TTS）"
 }
]
}
```

### 6.4 多选薄弱类型的处理

当用户选择多个薄弱类型时：
1. 生成一个**综合诊断**，覆盖所有选中类型
2. 练习句**按类型分组**，每组包含对应类型的练习
3. 输出 JSON 合并为：

```json
{
 "diagnosis": {
 "original_sentence": "...",
 "multi_type_analysis": {
 "pronunciation": { "issues":[...], "summary": "..." },
 "collocation": { "issues":[...], "summary": "..." },
 "structure": { "issues":[...], "summary": "..." },
 "implicature": { "issues":[...], "summary": "..." }
 }
 },
 "exercises": {
 "pronunciation":[ ...],
 "collocation":[ ...],
 "structure":[ ...],
 "implicature":[ ...]
 },
 "total_count": 10
}
```
